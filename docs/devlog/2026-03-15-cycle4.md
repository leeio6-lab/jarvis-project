## TOP 3 수정 내용

### 1위: "오늘 뭐했어?" — screen_texts 미활용 (4.8→6.7)

**원인:** `get_daily_summary()`가 `pc_activity`와 `app_usage`만 조회. screen_texts(화면 크롤링 데이터)를 전혀 포함하지 않음.

**수정:** `activity_analyzer.py`의 `get_daily_summary()`에 `screen_texts` 조회 추가. 중복 제거 후 `visited_sites` 필드로 반환. 오케스트레이터 도구 설명도 업데이트.

**결과:**
- 전: "오늘은 특별한 활동 기록이 없습니다"
- 후: "쿠팡, 네이버 부동산, 감사보고서 검색 등 웹사이트들을 방문하셨습니다"

### 2위: 캘린더 등록 도구명만 출력 (1→8)

**원인:** 오케스트레이터 `_try_text_fallback`에서 `tool_use_id: "fallback"`이 유효하지 않아 Claude API가 tool_result를 처리하지 못함. 또한 `create_calendar_event` 도구가 없어서 `get_upcoming_events`(조회)만 호출됨.

**수정:**
1. `_try_text_fallback`의 tool_result 방식을 user message로 변경 (tool_use_id 문제 회피)
2. `create_calendar_event` dry-run 도구 추가 (Google Calendar 연동 전 대비)
3. 오케스트레이터 라우팅 규칙에 "일정 등록" 추가

**결과:**
- 전: `get_upcoming_events` (raw tool name)
- 후: "내일 오전 10시에 분기결산 회의가 60분간 등록 준비되었습니다"
